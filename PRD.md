# to-dollar-index PRD

원화 자산 지표를 달러 가치로 환산해 원화/달러 시계열을 일대일 비교하는 개인용 웹 서비스.

## 1. 개요

- **이름**: to-dollar-index
- **한 줄 정의**: 주가·최저임금·부동산 등 원화 기준 지표를 달러로 환산해, 원화 상승이 실질 구매력 상승인지 시계열로 비교하는 도구.
- **핵심 기능(단 하나)**: 각 지표의 원화 가치 vs 달러 가치 일대일 시계열 비교.
- **첫 화면 첫 액션**: 최근 환율 변동 확인.
- **수익 모델**: 없음. 프로젝트성으로 개발 후 본인이 직접 사용하며 유지보수.

## 2. 문제 정의

원화 가치 하락 국면에서 한국 자산 지표(주가·최저임금·부동산)는 원화 기준으로만 발표된다. 달러로 환산하면 실질 상승이 아닐 수 있으나, 공식적으로 달러 환산 지표가 제공되지 않아 개인이 매번 직접 계산해야 한다.

크리티컬한 문제는 아니고 **반복적인 사소한 불편**이다. 이 불편을 한 번의 조회로 해소하는 것이 목표다.

## 3. 사용자

- **주 사용자**: 개발자 본인 + 지인 소수 (개인용).
- **핵심 가치**: 편의성(매번 계산할 필요 없음), 정확성(수정주가·직전 환율 폴백 등으로 신뢰 가능한 수치).
- 인증·계정 없음. 공개 읽기 전용 웹.

## 4. 유스케이스

1. **삼성전자 주식** — 최근 주가 상승이 달러 가치로는 얼마나 오른 것인지, 차트에서 구간을 인터랙티브하게 설정해 "이 구간에서 원화 +X%, 달러 +Y%"를 확인한다.
2. **최저임금** — 매년 오른 최저임금이 달러 기준 실질 구매력으로도 올랐는지 확인한다.
3. **부동산** — 서울 특정 구 평균 집값을 달러로 환산해, 원화 현금 흐름이 묶이는 부동산 투자의 타당성을 스스로 검증한다.

## 5. 기능 요구사항

### FR-1. 환율 차트 (첫 화면)
- 접속 시 최근 원/달러 환율 시계열 차트를 먼저 보여준다.
- 최근 변동(방향·폭)을 즉시 파악할 수 있게 한다.

### FR-2. 지표별 원화/달러 이중 차트
- 지표(주가·최저임금·부동산)를 선택하면 동일 시계열에 대해 **원화 값과 달러 환산 값을 함께** 표시한다.
- 달러 환산 값은 저장된 원화 원본에 조회 시점 환율을 적용해 계산한다(원본은 원화만 저장).

### FR-3. 구간 수익률 계산 (ECharts dataZoom)
- 사용자가 차트에서 구간을 인터랙티브하게 조정하면, 해당 구간의 첫/마지막 거래일을 기준으로 다음을 계산·표시한다.
  - `krwReturnPercent`: 원화 기준 구간 수익률(%).
  - `usdReturnPercent`: 달러 기준 구간 수익률(%).
  - `exchangeRateEffect`: 환율 효과(%p) = 달러 수익률 − 원화 수익률. (원화 약세 구간이면 음수 — "원화로는 올랐지만 달러로는 깎였다"를 한눈에)
- dataZoom 이벤트에 반응해 값이 실시간 갱신된다.

### FR-4. 데이터 수집 (배치)
- Python 배치가 하루 1회 지표 원본 데이터를 수집해 DB에 upsert한다.
- CLI 명령: `backfill`(과거 전체), `daily`(증분), `--dry-run`(쓰기 없이 검증).
- upsert 기반으로 **완전 멱등** — 같은 날 여러 번 돌려도 결과가 동일하다.

### FR-5. 휴장/결측 처리
- 특정일에 주가는 없고 환율만 있거나 그 반대인 경우, 달러 환산 뷰에서 **직전 환율로 폴백**해 값을 계산한다.
- **멱등의 한정**: upsert 멱등성은 *동일 소스 값* 가정 하에서 성립한다. 소스 값 자체가 변하는 경우(수정주가 소급 재계산, 실거래 정정)는 증분 수집으로는 과거 행이 갱신되지 않으므로 FR-6 재백필로 처리한다.

### FR-6. 전체 재백필 잡 (수정주가 basis drift 대응)
- 수정주가는 **조회 시점 기준으로 소급 계산**되므로, 미래에 새 액면분할·무상증자가 발생하면 증분 수집만으로는 과거 행이 옛 기준으로 남아 시계열이 불연속해진다.
- 이를 방지하기 위해 **월 1회 전체 재백필 잡**을 둔다: 전 구간을 재조회한 뒤 upsert로 현재 기준에 맞춰 과거 값을 재정렬한다.
- 재백필도 upsert 기반이므로 멱등하며, 값 변화가 없으면 결과가 동일하다.

## 6. 비기능 요구사항

- **정확성**: 수정주가 사용(액면분할 연속성), 직전 환율 폴백으로 결측일 처리. 수치가 사용자의 판단 근거가 되므로 정확성이 최우선.
- **멱등성**: 모든 수집은 upsert 기반. 재실행/재수집이 안전하며 자가 치유된다.
- **자동 복구**: 수집 실패일이 있어도 다음 실행에서 `max(price_date)+1`부터 증분 수집으로 자동 복구.
- **보안(RLS)**: 모든 base 테이블에 RLS ON + **anon은 SELECT-only 정책**, anon의 INSERT/UPDATE/DELETE grant는 회수. 쓰기는 service_role만. `daily_prices_with_usd` 뷰는 `security_invoker=on`으로 생성해 호출자(anon) 권한으로 base 테이블 정책이 적용되게 한다.
- **성능/비용**: 별도 API 서버 없이 Next.js Server Component가 Supabase를 fetch. ISR `revalidate 3600`으로 캐싱. 정적 데이터 특성상 실시간성 불필요.
- **운영 부담 최소화**: GitHub Actions cron으로 무인 수집. 개인 유지보수 가능한 수준의 단순 구조 유지.
- **복합 실패 체인 방지(keepalive)**: GitHub Actions는 레포에 **60일간 새 커밋이 없으면** 스케줄 워크플로가 자동 비활성화되고, Supabase 무료 티어는 **7일간 DB 활동이 없으면** 프로젝트가 일시정지된다. → **월 1회 전체 재백필 워크플로(FR-6)**가 실행 시 heartbeat 파일을 커밋(keepalive)해, 커밋 활동과 DB 활동을 동시에 유지함으로써 두 문제를 함께 방지한다.

## 7. 기술 스택

### 수집 (collector/)
- Python 배치, CLI(`backfill` / `daily` / `--dry-run`).
- 실행: GitHub Actions cron, **평일 KST 19시 일 1회**, 일일 종가 기준.
- 데이터 소스
  - **주가**: KIS 오픈API 국내주식 기간별시세(`FHKST03010100`, 수정주가).
  - **환율**: KIS 오픈API 해외주식 종목/지수/환율 기간별시세(`FHKST03030100`, `FID_COND_MRKT_DIV_CODE='X'`).
    - **리스크·대안**: 환율 TR에 환율 조회 시장구분('X')은 존재하나, 원/달러의 정확한 `FID_INPUT_ISCD`가 공식 문서에 미확정. 구현 시 **실계좌 dry-run으로 종목코드를 확정**하며, 확정 실패 시 **한국은행 ECOS API로 환율 소스를 교체**한다.
  - **최저임금**: 사용자 제공 정적 시드.
  - **부동산**: 국토부 아파트 실거래가 공공 API, 월 평균 집계.

### 데이터베이스 (supabase/)
- Supabase(PostgreSQL). 달러 환산은 저장하지 않고 **조회 시점 뷰에서 계산**.
- RLS: anon 읽기 전용, 쓰기 service_role 전용.

### 프론트 (web/)
- Next.js 15 (App Router) + TypeScript + Tailwind CSS + shadcn/ui.
- 차트: Apache ECharts (`echarts-for-react`, `dynamic import ssr:false`).
- **데이터 조회**: 별도 API 서버 없이 **Server Component에서 Supabase fetch + ISR `revalidate 3600`**으로 통일한다(브라우저에서 직접 조회하지 않음). 차트 컴포넌트는 서버가 내려준 데이터를 props로 받아 클라이언트에서 렌더링만 담당한다.
- 배포: Vercel (Root Directory = `web`).

## 8. 데이터 모델 개요

| 테이블 | 역할 | 핵심 필드(개요) |
|--------|------|-----------------|
| `indicators` | 지표 메타 (주가/최저임금/부동산 등) | id, type, name, code |
| `daily_prices` | 일별 종가 **원화 원본만** 저장 | indicator_id, price_date, close_krw |
| `exchange_rates` | 일별 원/달러 환율 | rate_date, usd_krw |
| `minimum_wages` | 연도별 최저임금 (정적 시드) | year, wage_krw |
| `real_estate_monthly` | 구별 월평균 실거래가 | region, year_month, avg_price_krw |

- **뷰 `daily_prices_with_usd`**: `daily_prices`에 당일 환율을 조인해 달러 환산 값을 계산. 당일 환율이 없으면 **직전 환율로 폴백**. 달러 값은 물리적으로 저장하지 않고 이 뷰에서 파생.

## 9. 단계별 범위

### Phase 1 — MVP
- 환율 차트 첫 화면.
- 국내주식(삼성전자 시드) 원화/달러 이중 차트.
- ECharts dataZoom 구간 수익률(원화%, 달러%, 환율효과%p).

### Phase 2 — 최저임금
- 연도별 최저임금 원화/달러 비교 (정적 데이터).

### Phase 3 — 부동산
- 서울 구별 월평균 원화/달러 비교.
- 국토부 아파트 실거래가 API 수집 포함.

## 11. 성공 지표

- **완결성**: 세 유스케이스(주가·최저임금·부동산)를 각 지표 화면에서 원화/달러 비교와 구간 수익률로 답할 수 있다.
- **편의성**: 매번 손으로 하던 달러 환산 계산이 한 번의 조회로 대체된다(본인이 실제로 계산기를 안 쓰게 됨).
- **정확성**: 수정주가·직전 환율 폴백 적용으로 결측일·분할일에도 값이 끊기거나 튀지 않는다.
- **무인 운영**: 수집 배치가 사람 개입 없이 일 1회 돌고, 실패해도 다음 실행에서 자동 복구된다.
- **지속성**: 개인이 부담 없이 유지보수 가능한 구조로, 개발 후 실제로 계속 사용된다.
